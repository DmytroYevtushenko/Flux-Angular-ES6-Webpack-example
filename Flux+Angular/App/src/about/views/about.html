In this project it was used angular 1.* with ECMAScript 6 syntax, FLUX architecture, Karma with Jasmine for unit testing and webpack.

For download node_modules rum cmd, change project directory and run command:
npm i


For run development\production build rum cmd, change project directory and run command:
SET NODE_ENV=development && webpack
SET NODE_ENV=production && webpack Or simply run build in Task Runner Exploler.

For run unit tests just use npm test command.

For a more comfortable work with webpack i used WebPack Task Runner. Download


On Cards tab was used simple custom dispatcher and added logs. It's bad for used in projects, but easy debugging and can help understand better flux architecture. Logs available only in development, because in production build Webpack compress scripts and styles, dont create source maps and delete logs.

On Comments tab was used already created dispatcher for angular https://www.npmjs.com/package/flux-angular.

If you have this issue when running karma test, try to comment this line: response.setHeader('Expires', (new Date(0)).toString()) in "node_modules/karma/lib/middleware/commonjs". It works for me.
